"엑셀 매크로 배우면 야근이 줄어든다"는 말, 반은 맞고 반은 함정입니다. 어디까지가 직접 할 수 있는 영역이고 어디부터 배보다 배꼽이 커지는지, 경계선을 그어드릴게요.

## 직접 해볼 만한 것: 매크로 기록 기능

엑셀엔 프로그래밍 없이 쓰는 **매크로 기록** 기능이 있습니다 (보기 → 매크로 → 매크로 기록). 내가 하는 작업을 녹화했다가 버튼 하나로 재생하는 방식이에요. 이런 일에 잘 맞습니다.

- 매번 같은 서식 적용 (열 너비, 색상, 필터, 정렬)
- 같은 구조의 표에서 매번 같은 계산 반복
- 반복적인 복사-붙여넣기 패턴

**한 시간만 투자해서 기록 기능을 익히면, 매일 10~20분짜리 단순 반복은 확실히 없앨 수 있습니다.** 여기까지는 꼭 직접 해보세요.

## 함정 구간: 어설픈 VBA 공부

문제는 그다음입니다. 기록 기능의 한계에 부딪히면 VBA(엑셀 프로그래밍)를 검색하게 되는데, 여기가 갈림길이에요.

- 인터넷 코드를 복사해 붙였는데 안 돌아가서 반나절을 씀
- 조건이 하나 바뀌면 다시 처음부터 헤맴
- 어렵게 완성해도 파일 구조가 조금만 달라지면 깨짐

**본업이 개발이 아니라면, 이 구간에서 쓰는 시간이 자동화로 아끼는 시간보다 커지기 쉽습니다.** 취미로 배우는 게 아니라면요.

## 전문가를 부르는 게 이득인 신호

1. **여러 파일·여러 시스템이 얽혀 있다** — 쇼핑몰 관리자에서 받아서 → 엑셀 정리 → ERP 입력, 같은 다단계 흐름
2. **웹에서 데이터를 가져와야 한다** — 가격·재고·순위 수집은 엑셀 밖의 영역입니다
3. **다른 사람도 써야 한다** — 나만 아는 매크로는 내가 없으면 멈춥니다. 실행 파일 + 사용 설명이 필요한 시점
4. **틀리면 안 되는 데이터다** — 정산·재고처럼 실수 비용이 큰 작업은 검증 로직까지 갖춘 도구가 필요합니다

이런 자동화는 보통 **수십만 원, 3~7일**이면 제작됩니다. 매일 30분짜리 작업이면 연 120시간 — 시급 2만 원으로 쳐도 연 240만 원이니, 계산은 명확한 편이에요.
